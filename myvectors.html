    While implementation of push_back() method of the vector if vector size(number of elements pushed into vector) is equal to capacity(number of elements can store into vector) of the vector then what we do is increasing the capacity of the vector by 1 this leads to the problems why? Stay Tune.

    For example, at capacity=1 to push an element create a new vector of capacity 2 and copy all the old array elements to new vector and at the end add the new element. At capacity=2, to push an element create a new vector of capcaity 3 and copy all the old vector elements to new vector and at the end add the new element. Simillarly, at n=n-1, if we want to push an elements create a new vector of capacity of 'n' and the same other stuff. After 'n' push operations the total time T(n) (number of copy operations) is proportional to 1 + 2 + ... + n = O(n*n)
    Lets improve the complexity by using vector doubling technique. If the vector capacity is full, create new vector of twice the size, and copy items. With this approach, pushing 'n' items takes time proportional to O(n) (not n*n), How?
    Lets us assume that initially we started with capacity=1 and moved till capacity=32. That means, we do the doubling at 1,2,4,8,16. At capacity=1, we do 1 copy operation, at capacity=2, we do 2 copy operations, and capacity=4, we do 4 copy operations and so on. By the time we reach n=32, the total number of copy operations is 1+2+4+8+16 = 31, which is approximately equal to 2n value(32). If we observe carefully, we are doing the doubling operation log(n)(base 2) times. For n push operations we double the vector capacity log(n) (base 2) times. That means, we will have log(n) terms in the expression below. The total time T(n) of a series of n push operations is proportional to
    1 + 2 + 4 + 8 + ... + (n/4) + (n/2) + (n) = n + (n/2) + (n/4) + (n/8) + ... + 4 + 2 + 1
    = n (1 + 1/2 + 1/4 + 1/8 + ... + 4/n + 2/n + 1/n)
    = n(2) = 2n = O(n)
    So a conclusion is after n push operations the total time T(n) (number of copy operations) is proportional to O(n).
